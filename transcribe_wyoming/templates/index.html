Speakers — Voice Recognition

Define people and record/upload voice samples. The longer and cleaner the sample (10–30 s recommended), the more reliable the recognition. Changes are picked up by the engine immediately.

Unrecognized Voices

Utterances whose voice did not match any known person wait here (grouped: one group = the same voice). Listen, check who is speaking and assign to a person — by default the whole group is assigned, so the profile immediately gets several samples. The same recordings can be assigned by the LLM pipeline via utterance_id from the transcription event.

Recognition Log

Recent transcriptions with the identification decision (who + confidence). Helps you judge whether the recognition thresholds are well tuned and catch mistakes.

Settings

How the Home Assistant pipeline receives information about who is speaking. Changes take effect from the next transcription (no restart needed).
The backup includes people, voice samples, roles and settings (excluding the unrecognized-voice buffer and the log). Restoring overwrites existing files.

API Endpoints

POST /inference — whisper.cpp compatible

POST /v1/audio/transcriptions — OpenAI compatible

GET /health — readiness and liveness check

Example (curl)

curl http://127.0.0.1:8580/inference \
  -H "Content-Type: multipart/form-data" \
  -F file="@audio.wav" \
  -F temperature="0.0" \
  -F response_format="json" \
  -F language="en"
